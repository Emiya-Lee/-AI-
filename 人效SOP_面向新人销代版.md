# 人效提升系统 SOP（面向新人销代版）

---

## 一、项目背景

### 1.1 为什么做这个项目

- **新人成长慢**：产品知识多、培训周期长、快速上手难
- **传统培训难闭环**：学了不代表会用，用了不代表有效
- **人效归因黑箱**：店效好了，不知道是产品强还是人强

### 1.2 我们怎么做

- 把「销售动作」数字化 → 「人效」可量化
- 把「人效数据」汇聚 → 「店效关系」可逼近
- 把「销售问题」实时反馈 → 「跟进动作」可闭环

### 1.3 项目目标

- **短期**：让新人快速上手，缩短成长周期
- **中期**：建立人效量化体系，证明培训动作有效
- **长期**：找到「人效→店效」的量化关系，用数据支撑决策

## 二、产品使用说明（面向新人销代）

### 2.1 小程序入口与注册

- **入口**：扫码进入微信小程序（主管推送链接/扫码）
- **首次登录**：填写姓名、所属门店、入职时间
- **登录后**：进入首页仪表盘，显示个人数据

### 2.2 记录一次销售（核心动作）

**适用场景**：每次完成一次顾客接待后填写

**填写步骤**：
1. 选择本次接待的机型
2. 填写接待时长（分钟）
3. 选择本次讲解了哪些知识点（多选）
4. 选择顾客关注了什么（多选）
5. 本次是否成交
6. 有何问题反馈（选填，文字描述）

**预计耗时**：3分钟/次

**示例**：
机型：85C12L | 接待时长：45分钟 | 讲解点：SQD技术、竞品对比、回音壁演示 | 顾客关注：价格、尺寸 | 成交：否 | 问题：顾客觉得比海信贵太多，没有对应话术应对

### 2.3 提交销售问题

**适用场景**：
- 产品知识盲区（不知道某个功能怎么讲）
- 竞品对比不会答（顾客问海信/创维怎么比较）
- 顾客反馈异常（产品出现异常反馈不知如何处理）

**提交方式**：小程序 → 「问题反馈」入口 → 选择问题类型 → 简述场景 → 提交

**跟进承诺**：系统负责人24小时内回复

### 2.4 查看自己的数据

- 个人成交率趋势（近30天）
- 讲解覆盖率（本周 vs 上周）
- 与门店/团队均值的对比

## 三、数据链路与量化逻辑

### 3.1 数据流向

```
新人销代（填写）→ 微信小程序
        ↓
   你的后台（汇总）
        ↓
   人效分析（量化）
        ↓
   店效关系（逼近）
```

### 3.2 人效怎么算

- **单次记录** → 个人成交率 / 讲解覆盖率
- **多组数据** → 个人能力画像
- **多人多组** → 门店人效基线

### 3.3 人效→店效关系如何逼近

| 阶段 | 数据量 | 分析方法 | 结论 |
|------|--------|---------|------|
| 初期 | < 100条 | 散点图相关性 | 讲解覆盖率↑的人，成交率是否也↑ |
| 中期 | 100-500条 | 相关系数R计算 | 量化相关性强度 |
| 后期 | > 500条 | 回归分析 | 「讲解每提升10%，成交率提升X%」 |

**核心原则**：样本量越大，关系越准确，数据越多越好

### 3.4 你在后台能看到什么

- 门店整体人效仪表盘（成交率/讲解覆盖率）
- 各销代能力画像分布（谁强谁弱）
- 实时问题提交列表（按紧迫性排序）
- 薄弱项高频词统计（哪些知识点最薄弱）

## 四、执行流程

### 4.1 新人使用流程

```
入职 → 扫码进入小程序 → 填写基本信息
       ↓
每次销售后 → 记录一次销售（3分钟）
       ↓
遇到问题 → 提交问题反馈
       ↓
定期查看 → 自己的能力画像
```

### 4.2 你的跟进流程

```
收到问题反馈
       ↓
判断类型：产品知识 / 话术技巧 / 竞品应对 / 其他
       ↓
评估紧迫性：
  紧急（影响成交）→ 当天跟进
  普通（知识盲区）→ 周度处理
       ↓
形成解决方案：
  → 回复销代（解决当前问题）
  → 更新培训内容（系统性改善）
       ↓
数据验证 → 下一期数据中看改善效果
```

### 4.3 数据汇入流程（你来操作）

- **零售数据**：每周从已有系统导出Excel → 一键导入分析后台
- **能力记录**：新人小程序实时汇入，无需手动操作
- **时间投入**：每周约15-30分钟

## 五、关键指标体系

### 5.1 个人指标（新人可在小程序查看）

| 指标 | 计算方式 | 参考标准 |
|------|---------|---------|
| 成交率 | 成交次数 ÷ 接待总次数 | 门店均值40%，优秀60%+ |
| 讲解覆盖率 | 实际讲解点数 ÷ 标准讲解点数 | ≥80%为合格 |
| 问题提交率 | 提交问题次数 ÷ 接待总次数 | 主动反馈意识体现 |

### 5.2 门店/团队指标（你在后台看）

| 指标 | 计算方式 | 用途 |
|------|---------|------|
| 门店成交率均值 | 所有销代成交率算术平均 | 店效基准线 |
| 人效离散度 | 销代间成交率标准差 | 判断能力差距 |
| 讲解率→成交率相关性 | 相关系数R | 验证人效→店效假设 |

## 六、里程碑与预期产出

| 阶段 | 时间 | 目标 | 交付物 |
|------|------|------|--------|
| 内测期 | 第1个月 | 10人以内部测试跑通 | 流程跑通，问题清单V1 |
| 数据积累期 | 第1-3个月 | 收集≥500条记录 | 人效基线初步建立 |
| 关系验证期 | 第3-6个月 | 样本量支撑统计分析 | 人效→店效量化报告（初稿） |
| 全面推广 | 第6个月起 | 全区新人销代覆盖 | 培训体系嵌入SOP |

## 七、分工与角色

| 角色 | 职责 |
|------|------|
| 新人销代 | 每次销售后记录+提交问题反馈 |
| 系统负责人（你） | 维护后台、跟进问题，分析数据、产出报告 |
| 主管 | 督促新人记录、配合培训组织 |

---

## 八、风险与应对

| 风险 | 应对措施 |
|------|---------|
| 新人不愿填写 | 主管督促+激励机制（记录次数排行/奖励） |
| 数据质量差（乱填） | 设置必填项+逻辑校验+抽样核查 |
| 问题反馈后跟进滞后 | SLA：紧急问题24h回复，普通问题周回复 |
| 人效→店效关系不显著 | 引入更多变量（客流、促销、竞品上样等） |
