# 人效提升系统 SOP（面向新人销代版）

---

## 一、项目背景

### 1.1 为什么做这个项目

- **新人成长慢**：产品知识多、培训周期长、快速上手难
- **传统培训难闭环**：学了不代表会用，用了不代表有效
- **人效归因黑箱**：店效好了，不知道是产品强还是人强

### 1.2 我们怎么做

- 把「销售动作」数字化 → 「人效」可量化
- 把「人效数据」汇聚 → 「店效关系」可逼近
- 把「销售问题」实时反馈 → 「跟进动作」可闭环

### 1.3 项目目标

- **短期**：让新人快速上手，缩短成长周期
- **中期**：建立人效量化体系，证明培训动作有效
- **长期**：找到「人效→店效」的量化关系，用数据支撑决策

## 二、产品使用说明（面向新人销代）

### 2.1 小程序入口与注册

- **入口**：扫码进入微信小程序（主管推送链接/扫码）
- **首次登录**：填写姓名、所属门店、入职时间
- **登录后**：进入首页仪表盘，显示个人数据

### 2.2 记录一次销售（核心动作）

**适用场景**：每次完成一次顾客接待后填写

**填写步骤**：
1. 选择本次接待的机型
2. 填写接待时长（分钟）
3. 选择本次讲解了哪些知识点（多选）
4. 选择顾客关注了什么（多选）
5. 本次是否成交
6. 有何问题反馈（选填，文字描述）

**预计耗时**：3分钟/次

**示例**：
机型：85C12L | 接待时长：45分钟 | 讲解点：SQD技术、竞品对比、回音壁演示 | 顾客关注：价格、尺寸 | 成交：否 | 问题：顾客觉得比海信贵太多，没有对应话术应对

### 2.3 提交销售问题

**适用场景**：
- 产品知识盲区（不知道某个功能怎么讲）
- 竞品对比不会答（顾客问海信/创维怎么比较）
- 顾客反馈异常（产品出现异常反馈不知如何处理）

**提交方式**：小程序 → 「问题反馈」入口 → 选择问题类型 → 简述场景 → 提交

**跟进承诺**：系统负责人24小时内回复

### 2.4 查看自己的数据

- 个人成交率趋势（近30天）
- 讲解覆盖率（本周 vs 上周）
- 与门店/团队均值的对比

## 三、数据链路与量化逻辑

### 3.1 整体数据流向

```
新人销代（填写）→ 微信小程序
        ↓
   你的后台（汇总）
        ↓
   人效分析（量化）
        ↓
   店效关系（逼近）
```

### 3.2 人效三维度量化体系

人效由三个维度综合构成，覆盖「讲解过程 → 能力考核 → 成交结果」全链路：

#### 维度一：录音讲解文件（自选型号话术）

- **数据来源**：新人自选一台机型，录制讲解视频/音频上传
- **AI分析内容**：话术结构、卖点覆盖、竞品提及、说服力评分
- **输出指标**：讲解质量得分（0-100分）
- **频次要求**：每周至少上传1条
- **权重建议**：30%（反映主动练习意愿+话术质量）

#### 维度二：AI考试演练分数（公司演练考核）

- **数据来源**：公司AI演练考试成绩 / 考试录音AI评分
- **分析内容**：答题完整度、产品知识准确性、应变能力
- **输出指标**：演练分数（0-100分）
- **频次要求**：每月至少完成1次公司演练考核
- **权重建议**：30%（反映标准化产品知识掌握程度）

#### 维度三：实际成交录音AI总结

- **数据来源**：成交后上传与顾客的真实讲解录音
- **AI分析内容**：讲解要点提取、薄弱环节识别、顾客决策关键点
- **输出指标**：成交录音质量得分 + 薄弱项标签
- **频次要求**：每次成交后建议上传（非强制）
- **权重建议**：40%（反映真实销售场景中的综合能力）

#### 人效综合得分计算

```
人效综合得分 = 维度一×30% + 维度二×30% + 维度三×40%
```

> 注：各维度权重可根据实际数据分布动态调整，初期以成交结果为重

### 3.3 店效双重验证体系

店效不仅看绝对值，还引入结构占比和区域对比，让数据更有说服力：

#### 维度一：结构机型销量占品类比

- **数据来源**：门店销售明细Excel
- **计算方式**：门店结构机型（如TCL高毛利机型）销量 ÷ 门店智屏品类总销量
- **用途**：衡量门店是否在推公司主推机型（执行到位率）
- **理想状态**：结构机型占比持续提升，说明主推方向落地

#### 维度二：门店区域销量比例

- **数据来源**：区域汇总数据
- **计算方式**：门店销量 ÷ 所属区域总销量
- **用途**：衡量门店在区域中的市场竞争力
- **理想状态**：区域占比提升，说明门店人效转化为店效

#### 店效综合系数

```
店效系数 = 成交率 × 结构机型占比 × 区域竞争系数
```

### 3.4 人效→店效关系如何逼近

| 阶段 | 数据量 | 分析方法 | 结论 |
|------|--------|---------|------|
| 初期 | < 300条 | 散点图相关性 | 人效得分↑的人，店效系数是否也↑ |
| 中期 | 300-1000条 | 相关系数R计算 | 量化相关性强度 |
| 后期 | > 1000条 | 回归分析 | 「人效每提升10分，店效系数提升X」 |

**核心原则**：样本量越大，关系越准确；三个维度相互验证，更全面

### 3.5 你在后台能看到什么

- 门店整体人效仪表盘（人效三维度得分分布）
- 各销代能力画像（雷达图：讲解质量/演练分数/成交录音得分）
- 实时问题提交列表（按紧迫性排序）
- 店效系数仪表盘（成交率×结构占比×区域系数）
- 薄弱项高频词统计（哪些知识点最薄弱）

## 四、执行流程

### 4.1 新人使用流程

```
入职 → 扫码进入小程序 → 填写基本信息
       ↓
每次销售后 → 记录一次销售（3分钟）
       ↓
每周 → 自选型号录制讲解音频上传（3分钟）
       ↓
每月 → 完成公司AI演练考核
       ↓
成交后 → 上传实际讲解录音（可选）
       ↓
遇到问题 → 提交问题反馈
       ↓
定期查看 → 自己的能力画像 + 三维度得分
```

### 4.2 你的跟进流程

```
收到问题反馈
       ↓
判断类型：产品知识 / 话术技巧 / 竞品应对 / 演练疑问 / 其他
       ↓
评估紧迫性：
  紧急（影响成交）→ 当天跟进
  普通（知识盲区）→ 周度处理
       ↓
形成解决方案：
  → 回复销代（解决当前问题）
  → 更新培训内容（系统性改善）
  → 针对性推送演练任务（补薄弱项）
       ↓
数据验证 → 下一周期看三维度得分是否改善
```

### 4.3 数据汇入流程（你来操作）

| 数据类型 | 来源 | 汇入方式 | 频率 |
|---------|------|---------|------|
| 录音讲解文件 | 新人上传 | 小程序实时 | 每周 |
| AI演练分数 | 公司演练系统 | Excel导入/系统对接 | 每月 |
| 实际成交录音 | 新人上传 | 小程序实时 | 随时 |
| 店效结构数据 | 门店销售明细 | Excel导入后台 | 每周 |

**时间投入**：每周约15-30分钟

## 五、关键指标体系

### 5.1 人效三维度指标（新人可在小程序查看）

| 维度 | 指标 | 计算方式 | 参考标准 |
|------|------|---------|---------|
| 维度一 | 讲解质量得分 | AI分析录音→质量评分（0-100） | 70分以上为合格 |
| 维度二 | 演练考核分数 | 公司AI演练系统评分（0-100） | 75分以上为合格 |
| 维度三 | 成交录音得分 | AI分析实际录音→得分（0-100） | 70分以上为合格 |
| 综合 | 人效综合得分 | 30%×维度一 + 30%×维度二 + 40%×维度三 | 75分以上为优秀 |

### 5.2 店效双重系数（你在后台看）

| 指标 | 计算方式 | 用途 |
|------|---------|------|
| 成交率 | 成交次数 ÷ 接待总次数 | 店效基础指标 |
| 结构机型占比 | 结构机型销量 ÷ 智屏品类总销量 | 主推方向执行率 |
| 区域竞争系数 | 门店销量 ÷ 区域总销量 | 区域市场竞争力 |
| 店效综合系数 | 成交率 × 结构占比 × 区域系数 | 店效综合得分 |

## 七、分工与角色

| 角色 | 职责 |
|------|------|
| 新人销代 | 每周上传讲解录音+每月完成演练考核+成交录音上传+问题反馈 |
| 系统负责人（你） | 维护后台、跟进问题、分析三维度数据、产出报告 |
| 主管 | 督促新人执行、配合培训组织、审核演练结果 |

## 八、风险与应对

| 风险 | 应对措施 |
|------|---------|
| 新人不愿上传录音 | 主管督促+激励机制（录音次数排行/奖励） |
| 演练分数注水 | AI分析+人工抽检双重验证 |
| 数据质量差（乱填） | 设置必填项+逻辑校验+录音时长门槛 |
| 问题反馈后跟进滞后 | SLA：紧急问题24h回复，普通问题周回复 |
| 人效→店效关系不显著 | 引入更多变量（客流、促销、竞品上样等） |
